What is this?

A tool to help teams make more strategic ABS challenge decisions without cognitively overloading players.

Why do we need it?

ABS challenge strategy is not about being certain you are right. Challenges are two-second decisions about inches at high speed. Uncertainty is built in. The goal is knowing when that risk is worth taking.

There is already substantial research on ABS challenge value, break-even points, and optimal challenge strategy. But a batter or catcher cannot turn an expected-value table into a real-time decision between pitches.

How will we do that?

By turning that research into actionable guidance:

  • Four simple challenge modes
  • Exportable/printable guide to use in the dugout.

Before each plate appearance, coaches use the guide to set the mode. Players do not need to calculate expected value in real time; they only need to know when the situation calls for challenging a questionable pitch.

The Four Challenge Modes

No-Brainers

Only unmistakable misses.

Full Count

Only 3-2 calls.

Deep Count

Two strikes or three balls.

Aggressive

Any close pitch.

How The Modes Are Derived

The model is built around win expectancy. ABS challenges are about winning this game, so the same missed call can matter more or less depending on the inning, score, outs, runners, count, and challenge inventory.

The inputs can change, this prototype even includes two different ways to estimate challenge cost, but the load-bearing question does not: is the upside of a successful challenge worth the downside of losing one? As expressed in the core equation:

Expected Value = p * V - (1 - p) * C

p is expected success rate of the challenging player. It is not whether the particular pitch is overturnable; we cannot expect players to calculate that in real time. It is the general challenging ability of the player. This tool embraces the ambiguity that players face.

The v1 defaults are the public league-wide averages: 47% for batters and 59% for catchers. If the user believes a particular player is a different p, those are adjustable in Model Settings.

V is the win probability swing between the umpire's call and the corrected call. The app uses Savant's count-aware Game Strategy Explorer endpoint for state values. Inning, half, score, outs, runners, and count all matter.

C is the opportunity cost of burning a challenge. The default v1 proxy is 5.5 win probability points times inventory, remaining-game, and current-WP multipliers. Cost model v1.5, "Shadow Zone," can also be selected in Model Settings. This estimates the volume of remaining Shadow Zone pitches, their average win probability value, and the expected chance they are overturned.

Built To Extend

The public version uses Baseball Savant as the win-probability source and exposes the main assumptions in Model Settings. The architecture is meant to be plug-in friendly: a team can swap in different success rates, challenge-cost assumptions, thresholds, or a private win-probability model without changing the player-facing challenge modes.

Default Derivations

The public model ships with reasonable defaults, not universal truths. They are meant to make the tool usable before a team plugs in its own player reads, challenge data, or win expectancy model.

Success Probability

p is the challenging player's expected success rate over time. It is not an estimate of whether a particular pitch is overturnable; that pitch-by-pitch calculation is exactly what the player cannot do in real time.

The default public inputs are 47% for batters and 59% for catchers. These are behavioral averages, so teams should adjust them when they have a better read on a specific batter, catcher, or staff.

Win Expectancy Value

V is the win probability swing between the umpire's call and the corrected call.

The live model uses Baseball Savant's count-aware Game Strategy Explorer as the public win expectancy source. That means inning, half, score, outs, runners, and count all matter.

Pitch-Quality Candidate Filter

The default future-opportunity filter is the Statcast Shadow Zone: roughly one baseball's width inside or outside the strike-zone edge.

For batters, that means called strikes in the Shadow Zone. For catchers, that means called balls in the Shadow Zone. Game situation is handled separately by win expectancy.

Conservative v1 Defaults

Conservative v1 is a deliberately simple opportunity-cost proxy. It starts with a lost-challenge baseline, then scales it by challenge inventory, remaining game, and current win probability.

C = 5.5 WP points x inventory factor x remaining game factor x current WP factor

The 5.5 win-probability point baseline is anchored around neutral, middle-game future challenge opportunities. The sampler looked at ball-versus-strike win expectancy swings across representative states, then used plausible future challenge-worthy spots rather than average pitches.

v1 Current Win Probability Factor

The model uses current team win probability instead of raw score margin because a five-run game is not always the same. Bases loaded with no outs is different from bases empty with two outs.

The curve is intentionally asymmetric. Preserving a challenge matters somewhat more when protecting a competitive or favorable position than when trying to climb back from a poor one, because a trailing team may never reach the future high-value spot.

Shadow Zone v1.5 Defaults

Shadow Zone v1.5 estimates the cost of losing a challenge more directly. Instead of starting from a fixed 5.5-point baseline, it asks how many future pitch-quality candidates are likely to remain and what those chances are worth.

C = expected future Shadow Zone candidates lost
    x blended future success rate
    x average future win-probability value
    x inventory premium

v1.5 uses a blended future success rate because the rest of the game can include both batter-side and catcher-side opportunities. A specific player success override affects the current recommendation; the future lineup component stays anchored to the broader public batter default unless a team replaces it.

Mode Thresholds

The model assigns the most aggressive mode that clears its evidence bar. Broader modes require more proof because they authorize more potential challenges.

ModeDefault Rule
No-BrainersFallback when Full Count does not clear.
Full CountMedian expected value of full-count opportunities is at least zero.
Deep CountMedian expected value is at least zero and at least 55% of sampled opportunities are positive value.
AggressiveMedian expected value is at least zero and at least 65% of sampled opportunities are positive value.

Developer Notes

A private win-expectancy model can replace the public Savant source as long as it returns count-level win probabilities for the same game state: inning, half, score, outs, runners, and count. The app then keeps using the same expected-value equation and mode thresholds.

For the printable guide, the fastest path is to precompute representative state tables and load them the same way this prototype loads app/data/guide-winexp.json. The live tab can call the same model directly if a team wants exact, internal win expectancy instead of the public endpoint.

How To Use

Live Recommendation

Use this when someone can enter the exact game state and get the current mode.

  1. Set the challenge side, score, inning, outs, runners, and challenges left.
  2. My Team is the side receiving the recommendation. For catcher recommendations, My Team is the fielding team.
  3. Give the player the mode for that plate appearance.

Create A Dugout Guide

Use this before the game when the staff needs a printed lookup book instead of an app in the dugout.

  1. Choose Batter or Catcher, add a player or group label, and set the expected success rate.
  2. Print or save the guide as a PDF. CSV is available for Excel-style review.
  3. In game, look up challenges left, inning half, score column, outs, and runners, then give the player the mode.
